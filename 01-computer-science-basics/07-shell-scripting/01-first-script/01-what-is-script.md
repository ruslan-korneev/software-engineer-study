# Что такое shell-скрипт

## Определение

**Shell-скрипт** - это текстовый файл, содержащий последовательность команд для shell (командной оболочки). Скрипт позволяет автоматизировать выполнение команд, которые иначе пришлось бы вводить вручную одну за другой.

## Зачем нужны скрипты

1. **Автоматизация** - выполнение повторяющихся задач без ручного ввода
2. **Воспроизводимость** - одинаковый результат при каждом запуске
3. **Документирование** - скрипт сам по себе является документацией процесса
4. **Сложные операции** - объединение множества команд в одну логическую единицу
5. **Планирование** - запуск задач по расписанию (cron)

## Простейший пример

```bash
#!/bin/bash

# Мой первый скрипт
echo "Привет, мир!"
```

Этот скрипт делает всего одну вещь - выводит текст на экран. Но даже такой простой скрипт демонстрирует основные элементы:
- Строка shebang (`#!/bin/bash`)
- Комментарий (`# Мой первый скрипт`)
- Команда (`echo "Привет, мир!"`)

## Shell vs Shell-скрипт

| Shell (интерактивный режим) | Shell-скрипт |
|----------------------------|--------------|
| Команды вводятся вручную | Команды записаны в файле |
| Выполняется по одной команде | Выполняются все команды последовательно |
| Результат виден сразу | Можно запускать многократно |
| Подходит для разовых задач | Подходит для автоматизации |

## Какой shell использовать

Самые распространенные shells для скриптов:

- **bash** (Bourne Again Shell) - наиболее популярный, установлен по умолчанию в большинстве Linux-дистрибутивов
- **sh** (Bourne Shell) - базовый POSIX-совместимый shell
- **zsh** (Z Shell) - расширенный shell с дополнительными функциями
- **dash** - легковесный POSIX-совместимый shell

Для начинающих рекомендуется использовать **bash**, так как:
- Широко распространен
- Много документации и примеров
- Обратно совместим с sh
- Имеет полезные расширения

## Где применяются shell-скрипты

- **Системное администрирование** - резервное копирование, мониторинг, управление пользователями
- **DevOps** - CI/CD пайплайны, деплой приложений
- **Обработка данных** - парсинг логов, преобразование файлов
- **Установка ПО** - скрипты установки и настройки
- **Повседневные задачи** - переименование файлов, организация каталогов

## Преимущества и недостатки

### Преимущества
- Не требует компиляции
- Прямой доступ к системным утилитам
- Легко изучить базовые концепции
- Портабельность между Unix-системами

### Недостатки
- Медленнее компилируемых языков
- Сложная обработка ошибок
- Ограниченные возможности для сложных структур данных
- Зависимость от установленных утилит
