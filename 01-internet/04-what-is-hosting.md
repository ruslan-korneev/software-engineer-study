# Что такое хостинг?

**Хостинг** — это услуга размещения сайта/приложения на сервере, доступном через интернет 24/7.

```
Твой код → Загружаешь на хостинг → Сайт доступен всем по домену
```

## Зачем нужен?

Домашний компьютер не подходит:
- Не работает 24/7
- Нестабильный интернет
- Нет статического IP
- Низкая скорость отдачи

Хостинг-провайдеры решают эти проблемы.

## Типы хостинга

| Тип | Что это | Для кого | Цена |
|-----|---------|----------|------|
| **Shared** | Много сайтов на одном сервере | Маленькие сайты, блоги | $3-10/мес |
| **VPS/VDS** | Виртуальный выделенный сервер | Средние проекты | $5-50/мес |
| **Dedicated** | Физический сервер целиком | Большие проекты | $100+/мес |
| **Cloud** | Масштабируемые облачные ресурсы | Любые проекты | По потреблению |
| **PaaS** | Платформа (без управления сервером) | Разработчики | Разная |

## Shared Hosting

```
┌─────────────────────────────────┐
│         Один сервер             │
├────────┬────────┬───────┬───────┤
│ Сайт A │ Сайт B │Сайт C │Сайт D │
└────────┴────────┴───────┴───────┘
```

**Плюсы:** дёшево, просто, не нужно администрировать
**Минусы:** соседи влияют на производительность, ограничения по ресурсам и софту

Примеры: Beget, Timeweb, Hostinger

## VPS / VDS (Virtual Private Server)

```
┌─────────────────────────────────┐
│      Физический сервер          │
├────────────────┬────────────────┤
│    VPS 1       │     VPS 2      │
│   (твой)       │    (чужой)     │
│  2 CPU, 4GB    │   4 CPU, 8GB   │
└────────────────┴────────────────┘
```

Виртуализация разделяет физический сервер на изолированные виртуальные машины.

**Плюсы:** root-доступ, гарантированные ресурсы, свобода настройки
**Минусы:** нужно администрировать самому (Linux, настройка сервера)

Примеры: DigitalOcean, Linode, Selectel, Timeweb Cloud

## Dedicated Server

Арендуешь **целый физический сервер**.

**Плюсы:** максимальная производительность, полный контроль
**Минусы:** дорого, сложное администрирование

Примеры: Hetzner, OVH, Selectel

## Cloud Hosting

Облачные провайдеры: AWS, Google Cloud, Azure, Yandex Cloud

**Особенности:**
- Платишь за использование (CPU-часы, трафик, хранилище)
- Мгновенное масштабирование (автоматически добавить ресурсы при нагрузке)
- Много дополнительных сервисов (базы данных, очереди, CDN, машинное обучение)
- Сложнее в освоении

**Сервисы AWS (для примера):**
- EC2 — виртуальные серверы
- S3 — хранилище файлов
- RDS — управляемые базы данных
- Lambda — serverless функции

## PaaS (Platform as a Service)

```
git push → Автоматический деплой → Сайт работает
```

Не нужно настраивать сервер — только код. Платформа сама:
- Устанавливает зависимости
- Запускает приложение
- Управляет масштабированием

Примеры: Heroku, Vercel, Railway, Render, Fly.io

## Serverless

Код выполняется по требованию, без управления серверами.

```
Запрос пришёл → Функция выполнилась → Ответ
        (платишь только за время выполнения)
```

Примеры: AWS Lambda, Cloudflare Workers, Vercel Functions

## Что выбрать для backend?

| Уровень | Рекомендация |
|---------|--------------|
| Учёба / pet-проект | VPS ($5-10/мес) — DigitalOcean, Timeweb |
| Стартап | PaaS (Railway, Render) или Cloud |
| Продакшен | Cloud с настройкой инфраструктуры |

## На что смотреть при выборе

- **Uptime SLA** — гарантия доступности
  - 99.9% = ~8.7 часов простоя в год
  - 99.99% = ~52 минуты в год
- **Локация** — сервер ближе к пользователям = меньше задержка
- **Поддержка** — скорость ответа, русский язык
- **Бэкапы** — автоматические резервные копии
- **Масштабируемость** — возможность увеличить ресурсы

## Связь домена и хостинга

1. Покупаешь домен у регистратора
2. Арендуешь хостинг (получаешь IP-адрес сервера)
3. В настройках домена указываешь DNS-запись: `example.com → 123.45.67.89`
4. Теперь домен ведёт на твой сервер
