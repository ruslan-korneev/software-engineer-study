# Deployment Patterns (Паттерны развёртывания)

Паттерны развёртывания определяют стратегии и подходы к выпуску, обновлению и управлению приложениями в production-среде. Эти паттерны помогают обеспечить надёжность, масштабируемость и удобство сопровождения современных распределённых систем.

## Содержание раздела

## Когда применять

| Паттерн | Сценарий использования |
|---------|----------------------|
| **Strangler** | Миграция legacy-систем, переход на микросервисы |
| **Static Content Hosting** | SPA, статические сайты, медиа-контент |
| **Twelve-Factor** | SaaS, облачные приложения, контейнеризация |

## Связанные темы

- **Blue-Green Deployment** - два идентичных окружения для zero-downtime деплоя
- **Canary Releases** - постепенный rollout на часть пользователей
- **Feature Flags** - управление функционалом без передеплоя
- **Immutable Infrastructure** - неизменяемая инфраструктура как артефакт
- **GitOps** - декларативное управление через Git
