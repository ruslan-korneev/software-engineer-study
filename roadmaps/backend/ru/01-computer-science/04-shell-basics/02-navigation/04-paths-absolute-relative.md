# Абсолютные и относительные пути

## Два типа путей

В Unix/Linux существует два способа указать расположение файла или директории:
- **Абсолютный путь** — от корня файловой системы
- **Относительный путь** — от текущей директории

## Абсолютный путь

**Абсолютный путь** начинается с корневой директории `/` и указывает полный маршрут до файла или директории.

### Характеристики:
- Всегда начинается с `/`
- Работает одинаково из любой директории
- Полностью описывает расположение файла

### Примеры абсолютных путей:
```bash
/home/user/documents/report.txt
/etc/nginx/nginx.conf
/var/log/syslog
/usr/bin/python3
```

### Использование:
```bash
$ cat /etc/passwd               # читаем файл по абсолютному пути
$ cd /var/log                   # переходим по абсолютному пути
$ ls /home/user/projects        # смотрим содержимое
```

## Относительный путь

**Относительный путь** указывает расположение относительно текущей рабочей директории.

### Характеристики:
- Не начинается с `/`
- Результат зависит от текущей директории
- Обычно короче абсолютного пути

### Специальные элементы относительных путей:
| Символ | Значение |
|--------|----------|
| `.` | Текущая директория |
| `..` | Родительская директория |
| `~` | Домашняя директория |

### Примеры относительных путей:
```bash
documents/report.txt      # файл в подпапке documents
../images/photo.jpg       # файл в соседней папке images
./script.sh               # файл в текущей директории
../../config.txt          # на два уровня вверх
```

## Сравнение на примере

Допустим, текущая директория: `/home/user/projects/app`

```
/
└── home/
    └── user/
        ├── documents/
        │   └── notes.txt
        └── projects/
            └── app/          ← мы здесь
                ├── src/
                │   └── main.py
                └── README.md
```

### Доступ к файлам:

| Файл | Абсолютный путь | Относительный путь |
|------|-----------------|-------------------|
| main.py | `/home/user/projects/app/src/main.py` | `src/main.py` или `./src/main.py` |
| README.md | `/home/user/projects/app/README.md` | `README.md` или `./README.md` |
| notes.txt | `/home/user/documents/notes.txt` | `../../documents/notes.txt` |

## Практические примеры

### Работа с текущей директорией (.)
```bash
$ pwd
/home/user/scripts

$ ./my_script.sh          # запустить скрипт из текущей папки
$ ls ./                   # то же что ls
$ cp file.txt ./backup/   # копировать в подпапку backup
```

### Переход на уровень выше (..)
```bash
$ pwd
/home/user/projects/app/src

$ cd ..                   # → /home/user/projects/app
$ cd ../..                # → /home/user/projects
$ ls ../tests             # содержимое соседней папки tests
```

### Комбинирование
```bash
$ pwd
/home/user/projects/app

$ cp src/main.py ../backup/        # из подпапки в соседнюю
$ cat ../../documents/notes.txt    # файл на 2 уровня выше
```

## Когда что использовать?

### Используйте абсолютный путь:
- В скриптах и конфигурационных файлах
- Когда путь фиксированный
- При работе с системными файлами
- Когда нужна однозначность

```bash
# В скрипте
LOG_FILE="/var/log/myapp.log"
CONFIG="/etc/myapp/config.yaml"
```

### Используйте относительный путь:
- При работе внутри проекта
- Для портативности (проект может быть в разных местах)
- Для краткости в интерактивной работе

```bash
# В проекте
$ cd ~/projects/myapp
$ python src/main.py           # относительный путь
$ cat config/settings.yaml     # относительный путь
```

## Проверка существования пути

```bash
# Проверить, существует ли файл/директория
$ ls /path/to/check
ls: cannot access '/path/to/check': No such file or directory

# В скрипте
if [ -d "/path/to/dir" ]; then
    echo "Директория существует"
fi
```

## Важные замечания

1. **Пути чувствительны к регистру** в Unix/Linux:
   ```bash
   /home/User   # не то же что
   /home/user
   ```

2. **Избегайте пробелов в путях** или экранируйте их:
   ```bash
   $ cd "My Documents"
   $ cd My\ Documents
   ```

3. **Не начинайте имена с дефиса** — это может конфликтовать с опциями команд

4. **`~` раскрывается только в начале пути**:
   ```bash
   ~/documents       # работает → /home/user/documents
   /home/~/test      # НЕ работает, ~ остаётся как символ
   ```
