## Что такое файловая система?

**Файловая система** — это способ организации и хранения файлов на диске. В Unix/Linux все файлы организованы в виде **иерархического дерева**, начинающегося с корня.

## Корневая директория

Корень файловой системы обозначается символом `/` (forward slash). Это самая верхняя директория, от которой берут начало все остальные.

```
/
├── bin/
├── etc/
├── home/
│   ├── user1/
│   └── user2/
├── tmp/
├── usr/
└── var/
```

## Основные директории Linux

| Директория | Назначение |
|------------|------------|
| `/` | Корневая директория |
| `/bin` | Основные исполняемые файлы (ls, cp, mv) |
| `/sbin` | Системные утилиты (для root) |
| `/etc` | Конфигурационные файлы системы |
| `/home` | Домашние директории пользователей |
| `/root` | Домашняя директория суперпользователя |
| `/tmp` | Временные файлы |
| `/usr` | Пользовательские программы и данные |
| `/var` | Переменные данные (логи, кэш, почта) |
| `/dev` | Файлы устройств |
| `/proc` | Виртуальная ФС с информацией о процессах |
| `/opt` | Дополнительное ПО |
| `/lib` | Системные библиотеки |

## Домашняя директория

Каждый пользователь имеет свою **домашнюю директорию** — личное пространство для файлов:

```bash
/home/username    # Linux
/Users/username   # macOS
```

Домашняя директория обозначается символом `~` (тильда):

```bash
$ echo ~
/home/username
$ echo ~root
/root
```

## Текущая рабочая директория

**Текущая директория** (working directory) — это место, где вы сейчас находитесь в файловой системе. Все относительные пути отсчитываются от неё.

```bash
$ pwd
/home/username/projects
```

## Специальные обозначения

| Символ | Значение |
|--------|----------|
| `/` | Корневая директория |
| `~` | Домашняя директория текущего пользователя |
| `.` | Текущая директория |
| `..` | Родительская директория |

## Просмотр структуры директорий

### Команда ls
```bash
$ ls /           # содержимое корня
$ ls /home       # содержимое /home
$ ls -la         # подробный список с скрытыми файлами
```

### Команда tree (нужно установить)
```bash
$ tree           # дерево текущей директории
$ tree -L 2      # только 2 уровня вложенности
$ tree /home     # дерево конкретной директории
```

Пример вывода:
```
.
├── documents/
│   ├── notes.txt
│   └── report.pdf
├── downloads/
└── projects/
    ├── app1/
    └── app2/
```

## Практический пример

```bash
$ pwd                    # где я?
/home/user
$ ls                     # что тут есть?
documents  downloads  projects
$ ls projects/           # что в projects?
app1  app2  app3
```

## Важно помнить

1. В Unix/Linux всё — это файл (даже директории и устройства)
2. Путь к файлу — это маршрут от корня или от текущей директории
3. Директории разделяются символом `/` (в отличие от Windows `\`)
4. Имена файлов чувствительны к регистру: `File.txt` и `file.txt` — разные файлы
