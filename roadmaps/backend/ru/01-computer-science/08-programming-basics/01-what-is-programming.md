## Что такое программирование?

**Программирование** - это процесс создания инструкций (программ), которые компьютер может выполнить для решения определённых задач. По сути, программист "объясняет" компьютеру, что и в каком порядке нужно делать.

### Ключевые понятия

**Программа** - это последовательность инструкций, написанных на языке программирования, которые компьютер выполняет для достижения определённого результата.

**Код (исходный код)** - текст программы, написанный программистом на языке программирования.

**Алгоритм** - это чёткая последовательность шагов для решения задачи. Алгоритм - это "рецепт", а программа - его реализация на конкретном языке.

## Алгоритмы

Алгоритм должен обладать следующими свойствами:

1. **Дискретность** - алгоритм состоит из отдельных шагов
2. **Определённость** - каждый шаг должен быть однозначно понятен
3. **Конечность** - алгоритм должен завершаться за конечное число шагов
4. **Результативность** - алгоритм должен приводить к результату
5. **Массовость** - алгоритм применим к классу задач, а не к одной конкретной

### Пример алгоритма: приготовление чая

```
1. Взять чайник
2. Налить воду
3. Поставить на плиту
4. Дождаться закипания
5. Насыпать заварку в чашку
6. Залить кипятком
7. Подождать 5 минут
```

### Пример алгоритма в коде (Python)

```python
# Алгоритм: найти максимальное число в списке
def find_max(numbers):
    if not numbers:
        return None

    max_value = numbers[0]  # Предполагаем, что первый элемент - максимальный

    for number in numbers:
        if number > max_value:
            max_value = number  # Нашли число больше - обновляем максимум

    return max_value

# Использование
my_numbers = [3, 7, 2, 9, 1, 5]
result = find_max(my_numbers)
print(f"Максимальное число: {result}")  # Выведет: 9
```

## Как компьютер понимает инструкции?

Компьютер "понимает" только машинный код - последовательности нулей и единиц. Но писать программы в нулях и единицах крайне неудобно, поэтому были созданы языки программирования.

### Уровни абстракции

```
Высокий уровень    Python, JavaScript, Ruby
        |          "Человеко-понятные" языки
        |
        v
Средний уровень    C, C++, Rust
        |          Ближе к "железу", но ещё читаемые
        |
        v
Низкий уровень     Ассемблер
        |          Практически прямое управление процессором
        |
        v
Машинный код       01001010 11010010...
                   То, что реально выполняет процессор
```

## Краткая история программирования

### 1840-е: Ада Лавлейс
- Написала первый в истории алгоритм для аналитической машины Чарльза Бэббиджа
- Считается первым программистом в истории

### 1940-е: Первые компьютеры
- ENIAC (1945) - программировался переключением проводов и тумблеров
- Программирование было физическим процессом

### 1950-е: Первые языки программирования
- **Fortran** (1957) - для научных вычислений
- **COBOL** (1959) - для бизнес-приложений
- **LISP** (1958) - для искусственного интеллекта

### 1970-е: Структурное программирование
- **C** (1972) - универсальный язык, "отец" многих современных языков
- **Pascal** (1970) - для обучения программированию

### 1980-е: Объектно-ориентированное программирование
- **C++** (1983) - C с поддержкой ООП
- **Objective-C** (1984) - для разработки под Apple

### 1990-е: Интернет-эра
- **Python** (1991) - простой и мощный
- **Java** (1995) - "напиши один раз, запускай везде"
- **JavaScript** (1995) - язык для веб-браузеров
- **PHP** (1995) - для веб-разработки

### 2000-е и далее
- **C#** (2000) - ответ Microsoft на Java
- **Go** (2009) - от Google, для серверов
- **Rust** (2010) - безопасный системный язык
- **Swift** (2014) - для экосистемы Apple
- **TypeScript** (2012) - JavaScript с типами

## Основные парадигмы программирования

### 1. Императивное программирование
Описываем **как** делать - пошаговые инструкции.

```python
# Императивный подход: сумма чисел
numbers = [1, 2, 3, 4, 5]
total = 0
for n in numbers:
    total = total + n
print(total)  # 15
```

### 2. Декларативное программирование
Описываем **что** нужно получить, а не как.

```python
# Декларативный подход (функциональный стиль)
numbers = [1, 2, 3, 4, 5]
total = sum(numbers)
print(total)  # 15
```

### 3. Объектно-ориентированное программирование (ООП)
Программа состоит из объектов, которые взаимодействуют друг с другом.

```python
class Dog:
    def __init__(self, name):
        self.name = name

    def bark(self):
        print(f"{self.name} говорит: Гав!")

my_dog = Dog("Шарик")
my_dog.bark()  # Шарик говорит: Гав!
```

### 4. Функциональное программирование
Программа - это композиция чистых функций без побочных эффектов.

```python
# Функциональный подход
from functools import reduce

numbers = [1, 2, 3, 4, 5]
squared = list(map(lambda x: x ** 2, numbers))
total = reduce(lambda a, b: a + b, squared)
print(total)  # 55 (сумма квадратов)
```

## Что нужно для начала программирования?

1. **Текстовый редактор или IDE**
   - VS Code (универсальный)
   - PyCharm (для Python)
   - WebStorm (для JavaScript)
   - Sublime Text, Vim, Neovim

2. **Интерпретатор или компилятор**
   - Зависит от выбранного языка
   - Python: установить python
   - JavaScript: Node.js или браузер
   - C/C++: gcc или clang

3. **Терминал (командная строка)**
   - Для запуска программ
   - Для работы с инструментами разработки

4. **Система контроля версий**
   - Git - стандарт индустрии

## Навыки хорошего программиста

- **Логическое мышление** - умение разбивать задачи на шаги
- **Внимательность к деталям** - одна опечатка может сломать программу
- **Терпение** - отладка занимает много времени
- **Постоянное обучение** - технологии быстро меняются
- **Умение читать документацию** - ключевой навык
- **Коммуникация** - объяснять идеи команде и понимать требования

## Резюме

- Программирование - это создание инструкций для компьютера
- Алгоритм - последовательность шагов для решения задачи
- Языки программирования делают код понятным для человека
- Существуют разные парадигмы: императивная, декларативная, ООП, функциональная
- Для начала нужны: редактор, интерпретатор/компилятор, терминал, Git
