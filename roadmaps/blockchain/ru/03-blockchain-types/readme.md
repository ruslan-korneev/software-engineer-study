## Классификация по виртуальной машине

### 1. EVM-совместимые (Ethereum Virtual Machine)

Самая распространённая категория. Используют стековую виртуальную машину Ethereum, что обеспечивает:
- Совместимость кода между сетями
- Переиспользование инструментов (Hardhat, Foundry, Remix)
- Единый язык разработки — Solidity

**Основные представители:**
- [Ethereum](./01-evm-based/01-ethereum/readme.md) — оригинальная платформа
- [Polygon](./01-evm-based/02-polygon/readme.md) — масштабирование через сайдчейны и zkEVM
- [BNB Chain](./01-evm-based/03-bnb-chain/readme.md) — высокая скорость, низкие комиссии
- [Другие EVM](./01-evm-based/04-other-evm/readme.md) — Avalanche, Fantom, zkSync, Base

### 2. L2-решения (Layer 2)

Работают поверх Ethereum, наследуя его безопасность:
- **Optimistic Rollups** — предполагают валидность, проверка через fraud proofs
- **ZK Rollups** — криптографическое доказательство каждой транзакции

**Основные представители:**
- [Arbitrum](./02-l2-blockchains/01-arbitrum/readme.md) — лидер TVL, поддержка Rust через Stylus
- [Optimism](./02-l2-blockchains/02-optimism/readme.md) — OP Stack, Superchain экосистема

### 3. TVM-совместимые (TON Virtual Machine)

Асинхронная акторная модель, оптимизированная для шардинга:
- Каждый контракт — независимый актор
- Динамическое шардирование
- Асинхронные межконтрактные вызовы

**Основные представители:**
- [TON](./03-tvm-based/01-ton/readme.md) — интеграция с Telegram, 900M+ пользователей
- [Everscale](./03-tvm-based/02-everscale/readme.md) — SMFT консенсус, Solidity++

### 4. Альтернативные сети

Собственные виртуальные машины с уникальной архитектурой:
- Новые языки программирования
- Инновационные консенсус-механизмы
- Специализированные оптимизации

**Основные представители:**
- [Solana](./04-alternative-networks/01-solana/readme.md) — PoH, параллельное исполнение, Rust
- [StarkNet](./04-alternative-networks/02-starknet/readme.md) — ZK-Rollup, Cairo, нативный AA

## Архитектурные паттерны

### Монолитные vs Модульные

```
Монолитный блокчейн (Solana):
┌─────────────────────────────────┐
│  Консенсус + Исполнение + DA    │
│          (всё в одном)          │
└─────────────────────────────────┘

Модульный стек (Ethereum + L2):
┌─────────────────────────────────┐
│         L2 (Исполнение)         │
├─────────────────────────────────┤
│      Ethereum (Консенсус)       │
├─────────────────────────────────┤
│    Celestia/EigenDA (DA)        │
└─────────────────────────────────┘
```

### Синхронные vs Асинхронные

**Синхронная модель (EVM):**
- Транзакция выполняется атомарно
- Все вызовы в одном блоке
- Проще для разработки

**Асинхронная модель (TVM):**
- Сообщения между контрактами
- Eventual consistency
- Лучше масштабируется

## Тренды 2024-2025

### 1. ZK-технологии
- ZK-Rollups становятся мейнстримом
- zkEVM обеспечивает совместимость с Solidity
- Приватные вычисления через ZK

### 2. Модульность
- Разделение слоёв (исполнение, консенсус, DA)
- Superchains и App-Chains
- Shared Sequencing

### 3. Account Abstraction
- Нативная поддержка в StarkNet, zkSync
- ERC-4337 на EVM-сетях
- Social Recovery и батчинг транзакций

### 4. Интероперабельность
- Кросс-чейн мосты
- Общие стандарты (LayerZero, Wormhole)
- Унификация пользовательского опыта
