# Инфраструктура блокчейна

Инфраструктура блокчейна — это набор инструментов и сервисов, которые позволяют разработчикам взаимодействовать с блокчейн-сетями. Понимание этих компонентов критически важно для создания надёжных децентрализованных приложений.

## Обзор раздела

### Зачем нужна инфраструктура?

Для взаимодействия с блокчейном приложению необходимо:
1. **Подключение к сети** — через ноду (свою или внешнюю)
2. **Отправка транзакций** — подпись и broadcast
3. **Чтение данных** — балансы, состояние контрактов, события

### Варианты инфраструктуры

```
┌─────────────────────────────────────────────────────────────┐
│                    Ваше приложение                         │
└──────────────┬──────────────────────────┬──────────────────┘
               │                          │
    ┌──────────▼──────────┐    ┌─────────▼─────────┐
    │  Клиентская         │    │   Node Provider   │
    │  библиотека         │    │   (Alchemy,       │
    │  (ethers.js,        │    │    Infura,        │
    │   web3.js, viem)    │    │    QuickNode)     │
    └──────────┬──────────┘    └─────────┬─────────┘
               │                          │
               └──────────┬───────────────┘
                          │
               ┌──────────▼──────────┐
               │   Блокчейн нода     │
               │   (Geth, Besu)      │
               └──────────┬──────────┘
                          │
               ┌──────────▼──────────┐
               │   Ethereum сеть     │
               └─────────────────────┘
```

## Компоненты инфраструктуры

| Компонент | Описание | Примеры |
|-----------|----------|---------|
| **Node Providers** | Готовые ноды как сервис | Alchemy, Infura, QuickNode |
| **Клиентские библиотеки** | SDK для работы с блокчейном | ethers.js, web3.js, viem |
| **Клиентские ноды** | Собственные ноды | Geth, Besu |

## Когда что использовать?

### Node Providers (NaaS)
**Подходит для:**
- Быстрого старта разработки
- MVP и небольших проектов
- Разработчиков без DevOps экспертизы

**Преимущества:**
- Не нужно управлять инфраструктурой
- Высокая доступность
- Дополнительные API и сервисы

### Собственные ноды
**Подходит для:**
- Enterprise решений
- Проектов с высокими требованиями к приватности
- Когда нужен полный контроль

**Преимущества:**
- Нет зависимости от третьих сторон
- Полная приватность запросов
- Нет лимитов на запросы

## Практические советы

### Для разработки
```javascript
// Используйте провайдеры для разработки
const provider = new ethers.JsonRpcProvider('https://eth-mainnet.g.alchemy.com/v2/YOUR_KEY');
```

### Для продакшена
- Настройте fallback между несколькими провайдерами
- Используйте WebSocket для real-time данных
- Добавьте кэширование для оптимизации
- Мониторьте использование и лимиты

```javascript
// Multi-provider setup для надёжности
const providers = [
  new ethers.JsonRpcProvider('https://eth-mainnet.g.alchemy.com/v2/KEY1'),
  new ethers.JsonRpcProvider('https://mainnet.infura.io/v3/KEY2'),
  new ethers.JsonRpcProvider('https://xxx.quiknode.pro/KEY3'),
];

const provider = new ethers.FallbackProvider(providers);
```
